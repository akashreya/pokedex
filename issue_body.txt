When processing the evolution chain for certain Pokémon (e.g., Tandemaus), the API returns the next evolutions species name (e.g., "maushold").
Currently, our code attempts to fetch Pokémon data using the endpoint:
https://pokeapi.co/api/v2/pokemon/maushold
However, this returns a 404 error.

**Root Cause:**
- The species name from the evolution chain does not always match the /pokemon/{name} endpoint in PokéAPI.
- The evolution chain provides a species.url (e.g., https://pokeapi.co/api/v2/pokemon-species/925/), which contains the correct Pokémon ID.

**Steps to Reproduce:**
1. Fetch the evolution chain for Tandemaus from PokéAPI.
2. Attempt to fetch Maushold data using /pokemon/maushold.
3. Observe the 404 error.

**Expected Behavior:**
The app should fetch Pokémon data using the correct ID (e.g., /pokemon/925), extracted from the species.url in the evolution chain.

**Proposed Solution:**
- Parse the Pokémon ID from the species.url in the evolution chain.
- Use this ID to fetch Pokémon data from /pokemon/{id} instead of relying on the species name.

**Example:**
- Evolution chain returns:
  "species": { "name": "maushold", "url": "https://pokeapi.co/api/v2/pokemon-species/925/" }
- Fetch Pokémon data from:
  https://pokeapi.co/api/v2/pokemon/925
